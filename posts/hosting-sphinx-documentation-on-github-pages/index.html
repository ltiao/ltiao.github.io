Static files (leading underscores)

By default, static files are outputted to directories with leading underscores, e.g. _static. This can be problematic with Github Pages where directories with leading underscores are ignored.

While we could modify Sphinx's default configuration (e.g. html_static_path), a more straightforward solution is to bypass preprocessing on Github Pages to support Sphinx HTML output properly, as outlined in Bypassing Jekyll on GitHub Pages.

This is made even simpler by the fact that ghp-import has a -n flag, which will automatically include the necessary file (.nojekyll) for us.

$ ghp-import -n -p $(BUILDDIR)/html
